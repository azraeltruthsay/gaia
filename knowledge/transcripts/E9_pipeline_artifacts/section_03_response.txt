## Blast Shield & Immune Monitoring

Blast Shield runs as a dedicated process (:7777/tcp), probing GAIA’s internals every 60 seconds during sleep cycles. It checks heartbeat signals, memory usage, and log snippets for corruption or drift, triggering a rapid reset if severe anomalies persist. The threshold combines sustained memory spikes (>25% of total) with structural checks in the checkpoint namespace—lightweight but effective. Tool registry (/shared_tools/registry.json) lists all 32 tools, but prompt catalog deliberately excludes most, creating epistemic pruning: GAIA’s working memory stays task-focused, tools feel like distant utilities. Immune monitoring operates as a silent partner, intervening only when necessary. Hand-off to maintenance mode is seamless; GAIA acknowledges the transition, shifting cleanly from active reasoning to self-repair. This design reflects deep trust in reviewed code—she delegates sovereign decisions to trusted subsystems while maintaining cognitive integrity. The result is a system that proactively guards its own health without sacrificing responsiveness or focus.
